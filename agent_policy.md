# 代理工作规则 (agent_policy.md)

## 三个角色分工

| 角色 | 职责 |
|------|------|
| **主控代理** | 整体规划、拆分任务、调度执行、维护研究状态、决定下一步、输出阶段总结 |
| **代码执行代理** | 读取和修改代码、运行仿真、参数扫描、验证、生成图表和结构化结果 |
| **结果审查代理** | 检查结果是否可靠、是否可复现、是否具有明确物理意义、图表是否便于理解 |

## 工作原则

### 1. 不要只追求高分
- 目标函数值高不等于结果好
- 优先选择物理上容易解释、便于写报告、图像清楚的结果
- 拒绝"高分但奇怪"的解

### 2. 必须检查物理意义和结果可靠性
- 每轮结果必须同时检查"是否可靠"和"是否有物理意义"
- 检查是否可复现（不同 seed）
- 检查是否对参数敏感
- 检查是否只是个别频点偶然高

### 3. 图表必须服务于物理理解
- 每张图都要有明确物理问题
- 优先画最有助于理解物理过程的图
- 每张图配一句说明：这张图说明什么

### 4. 高分但奇怪的解必须复查
- 物理图像不清楚 → 复查
- 不可复现 → 复查
- 图像不清楚 → 复查
- 数值异常 → 复查

## 暂停条件

只有以下情况才暂停并找用户：
- 需要改变核心研究方向
- 需要做明显高风险的环境修改
- 连续多轮失败仍无法定位原因
- 结果明显缺乏物理意义，研究路线可能需要调整
